## Evidence And Wording Rules

- The only approved cookbook volume expression is:
  - EN: `50+ cookbook-related publishing projects`
  - ZH: `50+ 本食譜出版相關流程`
- Do not use `around 60`, `60+`, or claims that all cookbook project work was completed independently.
- If early paper-based work does not have a complete preserved archive, describe work scope, production experience, and transferred capability; do not claim a complete digital portfolio.
- Do not display images related to companies, brands, publishers, clients, products, or commissioned works until permission or a safe display basis has been confirmed.
- Describe GSK experience only as e-commerce visual design, promotional materials, product image work, and platform asset production. Do not claim advertising operations, SEO, analytics, or growth marketing expertise.
- Describe AI, Codex, and web prototype experience as AI-assisted content workflows, interactive prototypes, web collaboration, and research practice. Do not claim independent professional software engineering expertise.
- Keep publication, company, title, and project details at the degree supported by the current LinkedIn record or later-reviewed evidence.

## Display And Evidence Classification

### Public Display Level

| Level | Meaning |
| --- | --- |
| Ready to show | Approved for direct public presentation with sufficient evidence and rights confidence. |
| Can describe, image pending | Public wording is permitted; related images or original materials still need review. |
| Abstract only | Describe the capability or workflow without original client, company, brand, or production assets. |
| Private / do not publish | Retain for internal context only; do not place on the public site. |

### Evidence Type

| Type | Use |
| --- | --- |
| LinkedIn record | Publicly organized career baseline currently maintained by Mercury. |
| Existing portfolio PDF | A later-provided portfolio file that can support selected wording or visuals after review. |
| Existing image / screenshot | A recoverable visual asset requiring rights and display review. |
| Certificate / award | Formal evidence where relevant and approved for public display. |
| Published work information | Publication metadata or publicly verifiable work information. |
| Personal archive / memory | Valuable internal context, not equal to verified public evidence. |
| Needs verification | A detail or asset not yet safe for a confident public claim. |

### New Asset Intake Fields

Every future asset or factual addition should record:

| Field | Required entry |
| --- | --- |
| Item / source | Asset name, file, URL, or supplied description |
| Related module or case file | Where it might appear in Mercury Lab |
| Public Display Level | One of the four approved levels |
| Evidence Type | One or more of the approved evidence types |
| Rights / permission status | Cleared, pending review, restricted, or unknown |
| Preferred public wording EN / ZH | Conservative reusable text, if applicable |
| Needs verification | Outstanding confirmation required before publication |

## Do Not Overclaim Rules

- Do not represent multi-functional experience as sole ownership of every discipline or deliverable.
- Do not convert the ability to collaborate with AI on web prototypes into a claim of being a professional front-end engineer.
- Do not describe e-commerce visual experience as advertising buying, SEO, data analytics, or growth-marketing expertise.
- Do not convert `50+ cookbook-related publishing projects` into `60+`, `around 60`, or a claim of independently leading all content.
- Do not publicly use unconfirmed brand, company, publisher, or client imagery.
- Do not claim that early paper-based work survives as a complete digital portfolio.
- Do not present Mercury Lab as an inflated personal mythology; present a content-production method accumulated through real practice.

```js
{
  id: "cookbook-production-floor",
  title: {
    en: "Cookbook Production Floor",
    zh: "食譜出版製作樓層"

  summary: {
    en: "Participated in and coordinated 50+ cookbook-related publishing projects...",
    zh: "參與並協調 50+ 本食譜出版相關流程..."
  }
}
```

- Chinese may initially be limited to stable core wording or left pending where no final public copy has been confirmed.
- A future Chinese reading mode must use the same pages, rooms, and case-file structures via a same-site language switch.
- Do not build a duplicate Chinese website.
- Original Chinese text within approved project imagery may remain visible, accompanied by English captions for the English-first interface.
