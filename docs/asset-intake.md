# Asset Intake Guide

Use this file when supplying works, links, imagery, or typefaces. Public usage is decided against [mercury-content-source.md](mercury-content-source.md).

## Where To Place Files

- Create `assets/intake/` for private review material not yet cleared for display; this path is ignored by Git.
- Create `public/assets/approved/` only after an item is confirmed safe for the website.
- Do not place unreviewed company, client, publisher, product, or commissioned imagery in public paths.

## Work Image Submission

- Preferred formats: `.jpg`, `.png`, `.webp` for review; original high-resolution files are welcome.
- Provide: item title, approximate year, related room/case file, creator/owner, public display permission, required credit, and any redactions.
- Mark status: `cleared`, `pending review`, `restricted`, or `unknown`.
- Examples needed later: cookbook covers, fiction/publication material, prototype screens, self-owned multimedia work.

## Link Submission

- Provide: URL, label to display, destination type, public/private status, and preferred section.
- Useful links: LinkedIn profile, portfolio, approved publications, contact destination, downloadable resume if cleared.
- Collaboration Protocol destinations remain placeholders until approved.

## Typeface Submission

- Provide: font name, source URL/file, available weights/styles, webfont files if licensed, and license/usage permission.
- Preferred web formats after approval: `.woff2`.
- Confirm whether web embedding and public portfolio-site use are permitted.
- Avoid adding fonts until license status and readability at body sizes are confirmed.
